⚠️ 注意事项

  1. 分支命名规范：

    • feature/ - 新功能
    • bugfix/ - 修复bug
    • hotfix/ - 紧急修复
    • release/ - 发布版本

  2. 提交信息规范：

    • feat: - 新功能
    • fix: - 修复bug
    • docs: - 文档更新
    • style: - 代码格式调整
    • refactor: - 代码重构

  3. 安全提醒：

    • 推送前先拉取最新代码
    • 重要操作前创建备份分支
    • 定期清理无用分支

💡 小贴士：使用 git status 随时查看当前工作区状态，这是最常用的命令之一！
